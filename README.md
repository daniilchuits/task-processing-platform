# Task Processing Platform

Сервис создает таски, которые сохраняются в psql и отправляются для асинхронной обработки воркерам через rabbitMQ. Сервис написан на Go. Запускается с помощью docker compose.

## Стэк
- Golang
- Docker
- PostgreSQL
- RabbitMQ

## Возможности

- Запуск с помощью Docker Compose
- Первичная регистрация пользователя
- Аутентификация и выдача JWT-token'а
- Создание задач с помощью HTTP-запросов
- Отправка полученных файлов с помощью брокера RabbitMQ
- Асинхронная обработка worker'ом TODO
- Занесение обновленных результатов в psql

## Запуск

1. Клонируем проект
2. В терминале переходим в директорию проекта
3. Устанавливаем переменные окружения
    .env-файлы есть в:
    - ./.env
    - ./services/notification-service/.env
    - ./services/tasks-service/.env
    - ./services/worker/.env
4. Запускаем с помощью `docker compose up --build`

## API
Порт 8080

### регистрация и логин
```text
регистрация - POST http://localhost:8080/auth/register
логин POST http://localhost:8080/auth/login
```

Пример тела запроса для регистрации и логина
```text
{
    "login": "user123", (логин >= 6 символов, больше валидации нет)
    "password": "123456" (пароль >= 6 символов)
}
```

В ответе при успешном логине, получаешь jwt-token. Он длится 2 часа и его необходимо прикрепить к запросу в headers. 
Пример заголовка:
```http
Authorization: Bearer <jwt-token>
```

После этого к каждому запросу в заголовки устанавливается токен.
Токен длится 2 часа!
В токене содержится время, до которого он валиден и ID пользователя, которое берется из таблицы со всеми регистрациями.

Пользователь может просматривать только свои записи, при занесении каждой в таблицу, присваивается user_id для этой записи.

## Обработка

Получение данных по всем загруженым файлам пользователя - `GET http://localhost:8080/task`

Получение данных по конкретному файлу - `GET http://localhost:8080/task/{id записи}`

Запостить новую запись - `POST http://localhost:8080/task` файлы через HTTP принимаются в теле запроса в поле `form-data`, чтобы опубликовать файл, ему обязательно нужно присвоить Key="file".
Файлы принимаются .txt, .jpg, .mp3, .csv, .pdf (с pdf не пошло) и .zip форматов.

Параллельно общей программе, воркеры достают из RabbitMQ файлы (пути к файлам), обрабатывают их и результаты вставляют в PostgreSql

Постишь запись, определяется формат файла (из доступных). Запись публикуется в таблицу `tasks` и отправляется с помощью rabbitMQ в очередь для воркеров. Воркеры работают ассинхронно с основным кодом `api`. Они обрабатывают файл и посылают через rabbitMQ статус файла, когда файл будет обработан (`task finished`/`task failed`). Статус в БД меняет сервис уведомлений, который и получает из rabbitMQ сообщения о статусе (и ошибке).
